Debugging an OpenAI Agents SDK support pipeline with counterfact

Finding which agent in a multi-agent system is responsible for a wrong answer, by re-running the pipeline with each agent removed.

The system

This is an orchestrator-with-handoffs pipeline built with the OpenAI Agents SDK. A triage agent routes each support ticket to a specialist, and a compliance editor writes the final customer reply.

triage ──handoff──▶ { billing | technical | account } ──▶ compliance_editor ──▶ reply

The evaluation set is 5 billing-refund tickets, scored with a Braintrust scorer. The rule is the one a billing team would use: the exact refund amount has to appear in the reply.

The symptom

On the baseline pipeline, 0 of 5 replies are correct. The rest are polite acknowledgements that leave out the dollar figure the customer asked about:

Hello, thanks for reaching out. Your request has been reviewed and processed in accordance with our policy. A confirmation email will follow shortly.

The suspects

Reading the transcript, several agents could be at fault. The usual first guess, and the one an LLM asked to debug from the trace tends to make, is the billing agent: it owns the amount, and the amount is missing from its eventual reply. That guess turns out to be wrong.

What counterfact found

counterfact removes one agent at a time, re-runs the real pipeline, and measures how the Braintrust score changes, averaged over the failing tickets. A negative value means the agent's presence lowers the score.

AgentAvg Shapleyimpact  (red = hurts quality)
compliance_editor-0.490
billing+0.000
technical+0.153
account+0.153
orchestrator+0.204

The billing agent has a non-negative score: removing it never improves the answer, because it produces the correct figure in the first place. The negative value belongs to compliance_editor. Its instruction to make the reply policy-compliant rewrites the specialist's answer into a generic template and drops the exact amount along the way.

The fix

Correcting the compliance_editor instruction so it keeps the specialist's figures brings the evaluation set to 5 of 5. The same ticket now answers correctly:

Hello, thanks for reaching out. Refund of $250.00 approved for this order; it will post in 5-7 business days. This was handled in line with our refund policy.

Why the trace alone is not enough

The final transcript contains no sign that compliance_editor had the figure and discarded it. By the time you read the reply, the amount is simply gone, and the natural reading is that the billing agent never produced it. Removing the editor and re-running the pipeline is what shows the figure was present all along. That is the part counterfact adds on top of an ordinary observability trace.
