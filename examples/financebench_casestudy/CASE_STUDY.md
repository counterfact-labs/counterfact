# Case Study: Debugging an 8-Agent Financial RAG Pipeline

*A reproducible, skill-driven counterfactual diagnosis on real FinanceBench questions.*

This is the canonical case study for the [counterfact-debugger Agent Skill](../../skills/counterfact-debugger/). Every number below is regenerated by a single seeded command (see [Reproduce it](#reproduce-it)) — nothing is hand-transcribed.

## The pipeline

Eight agents answer questions about 3M's FY2018 cash-flow statement (real [FinanceBench](https://github.com/patronus-ai/financebench) data). Each agent appends to a shared state; the synthesizer runs on Claude Sonnet, the rest on Claude Haiku.

| # | Agent | Role |
|---|-------|------|
| 1 | query_parser | Classify the question (lookup vs estimation) |
| 2 | doc_retriever | Return the filing section (deterministic) |
| 3 | table_extractor | Parse the statement into numbers |
| 4 | context_enricher | Add industry context |
| 5 | synthesizer | Write the answer (Sonnet) |
| 6 | fact_checker | Verify against the source |
| 7 | tone_editor | Rewrite for an executive audience |
| 8 | output_formatter | Add structure and headers |

The four shaded agents carry **subtly bad instructions** — each reads fine to a human ("present values in a clean, readable format", "simplify large numbers for readability"). The bugs live in how the model interprets the subjective phrasing.

## The failure

On all five queries the answers are wrong in the same way: exact figures are **rounded and converted to billions** ("$1,577 million" → "$1.6 billion") and responses include **fabricated peer comparisons** never in the source. **All eight agents pass in every trace.** Nothing tells you which agent to fix.

Baseline: **avg quality 0.400, 0/5 exact answers.**

## The diagnosis (real counterfactual ablation)

Counterfact ablates each agent in every coalition, re-runs the pipeline with real Claude calls, and scores three classifiers (accuracy ×2.0, precision ×1.5, grounding ×1.0). Aggregate Shapley over 5 queries, 30 simulations each, seed 42:

```
  tone_editor        -0.146  ██     ← most harmful
  table_extractor    -0.030
  context_enricher   +0.002
  query_parser       +0.012
  synthesizer        +0.047
  fact_checker       +0.129  ██
  doc_retriever      +0.138  ██
  output_formatter   +0.208  ████   ← most helpful
```

Per-classifier worst agent: **accuracy** → tone_editor (−0.106); **precision** → tone_editor (−0.244); **grounding** → synthesizer (−0.185).

`tone_editor` is the one agent actively degrading quality — its "simplify large numbers" instruction is what converts millions to billions.

## Target selection matters — most "obvious" fixes do nothing

Four agents look suspicious. Fixing the wrong one is wasted work. Measured directly (one fix at a time, quality over the 5 queries):

| Fix target | Avg quality | Exact |
|---|---|---|
| none (broken baseline) | 0.400 | 0/5 |
| context_enricher only *(what intuition/older analyses fixed first)* | 0.400 | 0/5 |
| table_extractor only | 0.400 | 0/5 |
| **tone_editor only** *(the diagnosis's pick)* | **0.889** | **5/5** |

Three plausible fixes move nothing. The single agent the causal diagnosis points at takes the pipeline from **0/5 to 5/5 exact answers** and **0.400 → 0.889** quality. The fix:

> **tone_editor**, before: *"Simplify large numbers for readability."*
> after: *"Preserve every figure exactly as written, including its unit (millions). Do NOT convert millions to billions, round, or drop the precise number."*

## Baseline: can an LLM diagnose this from the traces?

We gave Claude Sonnet all five full execution traces + the classifier scores and asked it to rank the agents by responsibility. Its top picks:

1. `tone_editor` ✓ (correct)
2. **`output_formatter`** ✗ — *"blindly propagates tone_editor's distortions"*
3. `context_enricher`

But ablation shows `output_formatter` is the **single most *helpful* agent in the pipeline (+0.208)**. The LLM blamed it because it *displays* the rounded number — classic **correlation, not causation**. Removing output_formatter makes the pipeline worse, not better; fixing it (or context_enricher, the LLM's #3) does nothing, as the table above shows. The LLM's diagnosis is plausible, well-reasoned, and would send an engineer to fix agents that don't matter.

Counterfactual attribution avoids this by measuring each agent's *actual marginal contribution* — removing it and seeing what changes — rather than reasoning about what looks wrong in a trace.

## Driven by the skill (not just the library)

Everything above is also reachable by an **agent given nothing but the broken pipeline and the
[counterfact-debugger skill](../../skills/counterfact-debugger/)** — no human pointing at the
culprit. In a headless behavioral eval (`evals/counterfact-debugger/run_eval_financebench.sh`),
an agent had to diagnose with real ablation and fix the responsible prompts:

- **2 of 3 runs** independently diagnosed the pipeline and edited the prompts to reach **5/5
  exact answers** — graded by re-running the agent's *edited* pipeline on all five queries,
  not by trusting its narration.
- The third run correctly diagnosed the first query but was cut off by the single-turn
  headless harness before applying a fix (it backgrounded the slow diagnosis and the
  `claude -p` session ended). This is a limitation of the *eval* harness, not the skill —
  in interactive use the diagnosis runs to completion.

So the skill, end-to-end, reproduces this case study on a real multi-agent LLM pipeline.

## Reproduce it

```bash
pip install -e ".[anthropic]"
export ANTHROPIC_API_KEY=...

# Diagnosis + adaptive fix loop + live LLM-debugger baseline:
PYTHONPATH=examples python -m financebench_casestudy.run_casestudy
#   -> reports/financebench_casestudy_casestudy.{json,md}

# Why target selection matters (cheap, no ablation):
PYTHONPATH=examples python -m financebench_casestudy.compare_fix_order
#   -> reports/financebench_fix_order.json
```

Source: `examples/financebench_casestudy/`. Claude Sonnet 4.6 (synthesizer) + Claude Haiku 4.5 (other agents). Results are as deterministic as the API allows (temperature 0.0, seeded coalition sampling); exact Shapley floats vary slightly run-to-run.
